Point!
部分 PDF 的標題、目錄行等文字，因製作方式限制，
整行會被視為單一文字單元。點擊時藍色框會覆蓋整行，
但實際查詢的是點擊位置最近的單字，結果不受影響。

DPI 建議（掃描版 PDF）：
150 DPI 以下：OCR 容易誤判，翻譯品質差
300 DPI：建議最低標準，OCR 準確率高
400–600 DPI：更佳，尤其小字、細節多的頁面
600 DPI 以上：檔案變大但 OCR 改善有限

【一、啟動方式　(ButterLayer.exe)　】

- 執行 ButterLayer.exe 啟動伺服器，瀏覽器會自動開啟 http://127.0.0.1:5000
- 伺服器視窗不可關閉，關閉後網頁即失效
- 若要同時翻譯兩份文件，請在瀏覽器開兩個分頁（網址相同）


【二、Gemini API Key】

- 每次使用前須在網頁輸入 Gemini API Key
- API Key 會自動儲存在瀏覽器，下次開啟會自動填入
- 若忘記 Key，請至 https://aistudio.google.com/app/apikey 取得


【三、模型選擇】

2026年5月更新:

預設模型：
  gemini-2.5-flash      → 翻譯速度與品質平衡（推薦）
  gemini-2.5-flash-lite → 最快、最便宜，品質略低
  gemini-2.5-pro        → 最高品質，費用較高

Point!：Google 會不定期新增或停用模型。
輸入 API Key 後，點選模型欄位旁的「抓取可用模型」，
即可自動更新為目前 Google 提供的最新模型清單。

模型驗證機制：
- 每次開始翻譯前，程式會先驗證所選模型是否可用
- 若模型已被 Google 停用（帳號限制），會立即顯示錯誤並從下拉選單自動移除，
  請重新選擇其他模型後再上傳翻譯


【四、翻譯速度慢 / API 過載】

翻譯中若網頁顯示「Gemini 伺服器繁忙，等待 XX 秒後重試…」，
主因為 Gemini 伺服器暫時過載，屬正常現象。

處理方式：
- 不需要任何操作，程式會自動重試，最多重試 4 次
- 重試全部失敗後，該模型進入 10 分鐘冷卻，
  程式會自動切換備援模型（依費用從低到高依序嘗試）繼續翻譯
- 若長時間無進展，可等 1～2 小時後用「繼續翻譯」功能接續


【五、翻譯中斷 / 尋找翻譯歷史紀錄】

- 每翻完一頁，結果會即時自動儲存至 ButterLayer 資料夾內的
  data\outputs\，不會因中斷而全部遺失

- 若翻譯失敗或中途關閉視窗，下次開啟 ButterLayer.exe 後：
  1. 主頁「最近翻譯記錄」會顯示「⚠ 未完成」
  2. 點選該記錄，再按「▶ 繼續翻譯」即可從上次完成的頁面接續
  3. 找不到記錄時，可點「查看全部歷史記錄」搜尋所有曾翻譯過的檔案

注意：以下情況會導致暫存消失，無法續翻：
  ・手動刪除 data\ 資料夾
  ・解壓縮新版本時覆蓋整個 ButterLayer\ 資料夾（建議先備份 data\）
  ・只複製 ButterLayer.exe 到別處，未一併帶走 data\ 資料夾
  ・格式化或更換硬碟


【六、多語言文件】

- 程式預設會將文件內所有語言（英文、日文、韓文等）一律翻譯成目標語言
- 若翻譯結果仍出現原文語言，屬於 Gemini 模型行為，可嘗試換用 gemini-2.5-pro


【七、多欄排版 PDF】

- 程式會自動偵測單欄、雙欄、三欄排版，並依正確閱讀順序（左→右）翻譯
- 最多支援四欄，超過四欄或不規則分欄可能仍有順序錯誤，屬正常限制


【八、大型 PDF 注意事項】

- 幾百頁的 PDF 翻譯時間可能長達數小時，請保持伺服器開啟
- OCR 掃描版 PDF 比原生文字版更耗記憶體與時間
- 建議：超過 100 頁的文件可分割後再上傳，避免單次任務過長


【九、字詞查詢功能】

- 翻譯完成後，在 PDF 左側點擊任意字詞可查詢意義
- 查詢使用較輕量的模型，通常比翻譯快
- 若出現「請先設定 Gemini API Key」，請確認 Key 已輸入


【十、翻譯覆蓋功能（Translation Overlay）】

翻譯完成後，可將中文翻譯直接疊加顯示在 PDF 頁面上，方便對照原版排版。

操作方式：
- 點選 PDF 面板上方的「翻譯覆蓋」按鈕，按鈕亮起表示已開啟
- 再次點擊即可關閉，回到原始 PDF 顯示
- 切換頁面時覆蓋會自動跟隨更新

適用範圍：
- 原生文字 PDF：依原始文字區塊位置精確對位
- 掃描版 PDF：依 OCR 偵測的段落邊界對位，位置接近原文段落

問題回報：
- 開啟翻譯覆蓋後，工具列會出現「問題回報」按鈕
- 點擊後自動下載整份文件的診斷資料（JSON 檔）
- 若覆蓋對位異常或翻譯顯示有誤，請附上此檔案以及翻譯文件回報開發者

Point!：
- 覆蓋為純顯示輔助，不影響下載的 Word 檔內容
- 掃描版 PDF 因 OCR 精度限制，位置對位可能與原文略有偏差


【十一、編輯畫面操作】

- 左右欄中間的分隔線可拖曳，調整原始 PDF 與翻譯結果佔據的螢幕比例。
- 左側 PDF 與右側翻譯結果皆有縮放控制（－ / 比例 / ＋）。
- 右側工具列最右邊可調整選取段落的字體大小與粗體：
  A－ 縮小、A＋ 放大、B 加粗。
- 右下角狀態列會同時顯示：
  PDF 第幾頁 / 總頁數、閱讀版 Word 第幾頁 / 總頁數、閱讀對照版 Word 第幾頁 / 總頁數。
  頁碼為依目前畫面與 Word 版面估算，實際下載後可能因 Word 版本、字型或印表機設定略有差異。


【十二、下載格式說明】

  閱讀版 Word       → 純翻譯文字，適合閱讀
  閱讀對照版 Word   → 一頁原始 PDF 圖片 + 一頁翻譯，交互對照
  OCR 版 Word       → 保留原版頁面排版位置的翻譯版本


【十三、常見錯誤訊息對照】

- 503 UNAVAILABLE / Gemini 伺服器繁忙
  Gemini 伺服器暫時過載。網頁會顯示等待倒數，程式自動重試，無需操作。
  重試耗盡後模型冷卻 10 分鐘，期間自動改用備援模型繼續翻譯。

- 模型「XXX」無法使用（已停用或帳號限制）
  所選模型已被 Google 停用或不支援目前帳號。
  該模型會自動從下拉選單移除，請重新選擇其他模型後再翻譯。
  可點「抓取可用模型」重新整理可用清單。

- 404 NOT_FOUND（模型名稱）
  同上，所選模型已停用。程式翻譯前會自動偵測並提示，通常不會在翻譯途中出現。

- All strings must be XML compatible
  翻譯或 OCR 文字含有特殊字元，程式已內建自動過濾。若持續出現請回報開發者。

- 處理失敗：（錯誤原因...）
  翻譯任務中途失敗。已完成的頁面均已自動存檔，
  可從主頁最近記錄點「繼續翻譯」接續，無需重新上傳。
